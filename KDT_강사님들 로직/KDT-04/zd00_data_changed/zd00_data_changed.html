Description: [D00] Data changed event sample 
*&---------------------------------------------------------------------*
*& Report ZD00_DATA_CHANGED  (데이터 체크)
*&---------------------------------------------------------------------*
* 1. PLANETYPE 및 SEATSOCC 열의 상태를 편집 가능으로 설정합니다.
*      테이블의 특정 열이 사용자가 편집할 수 있도록 설정합니다.
* 2. 기본 기능을 '변경만 가능'으로 제한 (선택 사항).
*      사용자가 새로운 행을 추가할 수 없도록 테이블 기능을 제한할 수 있습니다.
* 3. Enter 키를 누르면 이벤트 **DATA_CHANGED가 발생하도록 설정 (선택 사항).
*      기본적으로 사용자는 체크 아이콘을 사용하여 데이터를 검증할 수 있지만,
*      Enter 키로도 이벤트를 발생시킬 수 있게 할 수 있습니다.
* 4. 이벤트 DATA_CHANGED를 처리할 이벤트 핸들러를 정의하고 구현합니다.
*      테이블의 데이터가 변경되면 이벤트 핸들러가 이 이벤트를 처리할 수 있도록 구현합니다.
* 5. 테이블 MT_GOOD_CELLS를 순회(loop)하여 DDIC(딕셔너리) 정보를 기준으로 유효한 값을 확인합니다.
*      딕셔너리의 규칙을 기반으로 변경된 테이블의 각 셀 값을 확인합니다.
* 6. 검증 사이클 내에서:
*  a. GET_CELL_VALUE 메서드를 사용하여 새로운 셀 값을 가져와서 검증합니다.
*     (이 경우는 SEATSOCC 필드).
*     사용자가 입력한 새 값을 가져와서 확인합니다.
*  b. 값이 유효한 경우 다른 셀의 값을 변경할 수 있습니다.
*     검증 결과 값이 유효하면 다른 셀의 값을 변경할 수 있습니다.
*  c. 값이 유효하지 않으면 애플리케이션 로그에 기록을 생성합니다.
*     값이 유효하지 않으면 오류 메시지를 애플리케이션 로그에 남깁니다.
*  d. 이 검증 사이클에서 변경되지 않은 이전 값을 접근하려면 출력 테이블 GT_OUTTAB을 사용합니다.
*     검증 과정에서 변경되지 않은 이전 값을 확인하려면 출력 테이블에서 값을 가져옵니다.
* 7. 오류가 발생했을 경우 애플리케이션 로그를 표시합니다.
*    오류가 발생하면 애플리케이션 로그를 사용자에게 보여줍니다.
